# requirements.txt — Pinned dependency versions for reproducible installs
#
# PYTHON VERSION: 3.10 or higher (3.11 recommended for maximum compatibility).
#
#   MINIMUM 3.10 — three independent reasons:
#     1. numpy==2.2.x and scipy==1.15.x dropped Python 3.9 support.
#     2. `match` statements (structural pattern matching) in st-plot.py and
#        st-voice.py are Python 3.10+ syntax — 3.9 will raise a SyntaxError.
#     3. Python 3.9 itself is EOL (October 2025).
#
#   MAXIMUM — none.  TTS packages (soundfile, yakyak, websockets) ship wheels
#     for Python 3.10 through 3.13 on macOS ARM64.  soundfile's wheel is
#     tagged py2.py3-none (any Python version); yakyak is py3-none-any;
#     websockets ships per-version wheels for cp39–cp313.  No ceiling exists.
#     3.11 is recommended because it is the version used in active development
#     and most thoroughly tested, not because of any wheel restriction.
#
#   NO-TTS OPTION — to skip audio entirely, use requirements-no-tts.txt.
#     st-speak, st-voice, and st-prep --mp3/--all will not be available,
#     but all other commands work normally.
#
#   Create the venv with:  python3.11 -m venv .venv
#
# VERSIONING STRATEGY:
#   Direct dependencies are fully pinned (==) for stability and reproducibility.
#   Transitive dependencies are intentionally NOT pinned here — pip resolves them.
#   This avoids conflicts when direct deps (e.g. google-genai) update their own
#   transitive requirements. Run `pip freeze > requirements.frozen.txt` after
#   install if you need a fully-reproducible snapshot.
#
# MAINTENANCE:
#   - Run `pip list --outdated` periodically to check for updates.
#   - Run `pip-audit` or use GitHub Dependabot to scan for CVEs automatically.
#   - After testing an upgrade, update the version here and commit.
#
